서버 제출 테스트1

    이름과 나이를 입력 받아서 서버(JAVA 코드)로 제출하겠다.

    *준비물*
    
    1) form 태그 : 내부에 작성된 input값을 지정된 경로로 제출
    - action: 제출할 경로 
    - method: 제출 방법 (get/post) 기본방식 : GET방식으로 제출됨
    
    제출 == 클라이언트가 서버로 요청

    2)input 태그 : 값을 입력하는 양식 제공 
    - name (*중요*): 서버로 데이터 제출시에 key 역할 
    - value or 입력된 값 : 서버로 제출시 value 역할

    

      **GET : GET을 통한 요청은 URL 주소 끝에 파라미터로 포함 되어 전송되며, 이 부분을 query String 이라고 부른다.
      방식은 URL끝에 "?"를 붙이고 그 다음 변수명1=값1&변수명2=값2....
      ex) www.example.com/?name1=value1&name2=value2.....
      ->서버에서는 name1과 name2라는 파라미터 명으로 각각 value1과 value2 값을 전달 받을 수 있다.


      [장점]
      -단순함
      -캐싱 가능 (주소저장 == 북마크, 즐겨찾기, 주소공유)

      [단점]
      -요청 data가 url(주소)에 그대로 노출되기 때문에 보안에 취약
      -요청 data에 길이 제한이 있다. (다량의 많은 데이터를 보낼순 없다.)
